Five experiments tested in this session. All regressed; best val_bpb unchanged at **1.0387**.

### Experiment 1: Label smoothing 0.05

**Rationale:** Different regularization mechanism from softcap (which clamps logits) and weight decay (which regularizes weights). Label smoothing redistributes probability mass from target to non-target classes, potentially improving generalization.

|--------|-------|

| val_bpb | **1.2263** ✗ (prev best: 1.0387) |

| Steps | 1443 |

| MFU | 15.12% |

| VRAM | 26,895 MB |

| Tokens | 189.1M |

**Status: DISCARD** — Massive regression (+0.1876). Label smoothing fundamentally changes the loss landscape in this architecture. Training loss was ~3.43 vs typical ~2.93. Likely interacts poorly with the logit softcap (both redistribute probability mass, causing over-smoothing). Also increased VRAM by 4 GB for unknown reasons.

### Experiment 2: Gradient clipping max_norm=1.0

**Rationale:** Stabilize Muon optimizer gradients, especially in early training. Could prevent occasional large gradient spikes from destabilizing the Newton-Schulz orthogonalization.

| Metric | Value |

|--------|-------|

| val_bpb | **1.0466** ✗ (prev best: 1.0387) |

| Steps | 1482 |

| MFU | 15.53% |

| Tokens | 194.2M |

**Status: DISCARD** — val_bpb regressed by 0.0079. The Muon optimizer's Newton-Schulz orthogonalization already implicitly normalizes gradients (projects onto orthogonal matrices). External gradient clipping is redundant and may actually interfere with the optimizer's internal gradient processing.

### Experiment 3: Faster Muon momentum warmup (300→100 steps)

**Rationale:** Currently 22% of training is spent ramping momentum from 0.85 to 0.95. Shorter ramp lets optimizer reach full momentum sooner, maximizing effective training time at peak momentum.

| Metric | Value |

|--------|-------|

| val_bpb | **1.0473** ✗ (prev best: 1.0387) |

| Steps | 1452 |

| MFU | 15.21% |

| Tokens | 190.3M |

**Status: DISCARD** — val_bpb regressed by 0.0086. The slow 300-step ramp serves an important role: it allows the optimizer to explore more broadly before committing to high-momentum directions. Faster ramp may cause premature convergence to suboptimal valleys.

### Experiment 4: Scheduled EMA decay (0.95→0.999 over training)

**Rationale:** Adaptive EMA: early training has volatile weights (lower decay = more responsive to rapid changes), late training has stable weights (higher decay = more smoothed average). Replace fixed EMA_DECAY=0.99.

| Metric | Value |

|--------|-------|

| val_bpb | **1.0437** ✗ (prev best: 1.0387) |

| Steps | 1553 |

| MFU | 16.29% |

| Tokens | 203.6M |

**Status: DISCARD** — val_bpb regressed by 0.0050. Note: this run got a fast GPU (1553 steps, 16.29% MFU — comparable to session 15's 1560 steps/16.35% MFU that produced the 1.0387 best). Despite the throughput advantage, EMA scheduling performed worse. Starting too low (0.95) means early EMA weights carry too much old information, and ending too high (0.999) means the final steps barely contribute. Fixed 0.99 provides a better balance.

### Experiment 5: Increase VE gate channels 32→64

**Rationale:** Value embedding gate uses first 32 channels of input to decide mixing strength per head. Using 64 channels gives a richer input signal for more informed gating decisions. Negligible parameter increase.

## Analysis

- All five experiments regressed, continuing the pattern from sessions 31-33
- The model is deeply optimized for this time budget — nearly every dimension has been explored
- Session 15's best (1.0387) likely benefited from a favorable GPU allocation (16.35% MFU)
- Experiment 4 got a similarly fast GPU (16.29% MFU) but still didn't beat 1.0387, confirming that fixed EMA=0.99 is well-calibrated
- Label smoothing is catastrophically bad with softcap — they both redistribute probability mass and the combination over-regularizes
- The Muon optimizer has strong built-in gradient management (Newton-Schulz orthogonalization), making external gradient clipping unnecessary
- GPU variance: runs at 15-15.5% MFU produce val_bpb in the 1.045-1.047 range; runs at 16%+ MFU produce 1.039-1.044

## Key confirmed values this session
