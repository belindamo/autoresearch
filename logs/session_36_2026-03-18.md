## Experiments

Six experiments tested in this session. All regressed; best val_bpb unchanged at **1.0387**.

### Experiment 1: Constant weight decay (remove warmdown decay)

**Rationale:** Currently WD decays linearly from 0.05 to 0 during the warmdown phase (last 67% of training). Keeping WD constant at 0.05 throughout training maintains regularization during the LR annealing phase, potentially helping convergence.

| Metric | Value |

|--------|-------|

| val_bpb | **1.0460** ✗ (prev best: 1.0387) |

| Steps | 1478 |

| MFU | 15.49% |

| VRAM | 22,806 MB |

| Tokens | 193.7M |

**Status: DISCARD** — Regressed by 0.0073. Despite a fast GPU allocation (15.49% MFU, 1478 steps), constant WD hurt. The warmdown WD decay is essential — maintaining full regularization during LR annealing interferes with convergence. The model needs WD to decrease alongside LR so weights can settle into their final values without being pulled toward zero.

### Experiment 2: DEVICE*BATCH*SIZE 64 → 32 (grad*accum*steps=2)

**Rationale:** Test whether BF16 gradient accumulation precision effects help by splitting each step into two micro-batches. Also reduces peak VRAM usage.

| Metric | Value |

|--------|-------|

| val_bpb | **1.1073** ✗ (prev best: 1.0387) |

| Steps | 1298 |

| MFU | 13.59% |

| VRAM | 11,854 MB |

| Tokens | 170.1M |

**Status: DISCARD** — Catastrophic regression (+0.069). While VRAM halved (22.8→11.9 GB), the overhead of two forward-backward passes per step reduced throughput by 12% (170M vs typical 183M tokens). The model saw significantly less data, causing severe underfitting. Training loss at end (~3.08) much higher than typical (~2.93). Gradient accumulation is compute-wasteful for this architecture.

### Experiment 3: Parallel attention+MLP (PaLM-style blocks)

**Rationale:** Compute attention and MLP on the same normalized input instead of sequentially. This is used in PaLM and other large models. Reduces one normalization and could change training dynamics.

**Status: DISCARD** — Regressed by 0.026. Parallel blocks hurt because the MLP no longer sees the attention output, reducing the model's ability to compose attention patterns with feedforward processing. Training loss at end (~2.98) higher than typical (~2.93). The sequential block design is important for this model's representational capacity at 8 layers.

### Experiment 4: Adam beta2 0.95 → 0.90 (embedding/lm_head optimizers)

**Rationale:** Faster variance adaptation in Adam could help for short training runs. The second moment estimate adapts more quickly to gradient variance changes, potentially improving optimization dynamics for embeddings and lm_head.

| Metric | Value |

|--------|-------|

| val_bpb | **1.0548** ✗ (prev best: 1.0387) |

| Steps | 1328 |

| MFU | 13.91% |

| VRAM | 22,806 MB |

| Tokens | 174.1M |

**Status: DISCARD** — Regressed by 0.016. At this MFU (~14%), baseline typically gives 1.044-1.050, so 1.0548 is slightly worse than expected. The faster variance adaptation is too noisy — beta2=0.95 provides sufficient smoothing for stable embedding/lm_head optimization. Adam beta2 map: 0.90→1.0548, **0.95→best**.

### Experiment 5: QK rescaling q*sqrt(head_dim) after normalization

**Rationale:** After QK-norm, both Q and K have unit RMS. FA3's default 1/sqrt(head*dim) scaling then produces very small attention logits (~0.09), leading to near-uniform attention distributions. Multiplying Q by sqrt(head*dim) compensates for this, giving attention logits that are raw dot products of unit-RMS vectors (range ~[-1, 1]).

| Metric | Value |

|--------|-------|

| val_bpb | **1.0872** ✗ (prev best: 1.0387) |

| Steps | 1451 |

| MFU | 15.21% |

| VRAM | 22,806 MB |

| Tokens | 190.2M |

**Status: DISCARD** — Large regression (+0.049) despite fast GPU (15.21% MFU). The sharper attention breaks the model's distributed attention patterns. The "near-uniform" attention from QK-norm + 1/sqrt(d) scaling is actually a feature, not a bug — it prevents entropy collapse and lets the model learn nuanced attention patterns through directional similarity rather than magnitude. Training loss (~3.04) much higher than typical, confirming the model can't learn effectively with overly sharp attention.

### Experiment 6: Standard weight decay (remove cautious masking)

**Rationale:** Replace cautious weight decay (only applied when gradient agrees with parameter direction) with standard weight decay (always applied). Simplifies the optimizer by removing the masking logic.

| Metric | Value |

|--------|-------|

| val_bpb | **1.0538** ✗ (prev best: 1.0387) |

| Steps | 1353 |

| MFU | 14.17% |

| VRAM | 22,806 MB |

| Tokens | 177.3M |

**Status: DISCARD** — Regressed by 0.015. At 14% MFU, baseline gives ~1.044-1.050, so 1.0538 is marginally worse. Standard WD over-regularizes by pulling weights toward zero even when gradients want them to grow. Cautious masking is important because it allows the optimizer to grow weights freely when needed while still regularizing when gradient and weight agree on the direction toward zero. Training loss (~2.95) slightly higher than typical.

## Analysis

- All six experiments regressed, continuing the pattern from sessions 31-35 (now 30+ consecutive regressions across 6 sessions)
- The model remains deeply optimized at **1.0387** for this GPU/time budget
- **WD warmdown decay is essential**: Constant WD (+0.007) interferes with LR annealing convergence
- **Gradient accumulation is wasteful**: 12% throughput loss from double fwd/bwd passes causes catastrophic underfitting
- **Sequential blocks outperform parallel**: MLP-seeing-attention-output is important for 8-layer model capacity
- **Adam beta2=0.95 is optimal**: Faster adaptation (0.90) too noisy for embedding/lm_head optimization
- **QK-norm scaling is well-calibrated**: The "soft" attention from 1/sqrt(d) is a feature; sharpening breaks learning
- **Cautious WD masking helps**: Allowing weights to grow freely when gradient wants growth is important

## Key confirmed values this session

- WD warmdown decay: **linear decay** >> constant (decay essential for convergence)
- DEVICE*BATCH*SIZE: **64** >> 32 (grad_accum overhead kills throughput)
- Block structure: **sequential** >> parallel (MLP needs attention output)
- Adam beta2: 0.90 < **0.95** (for embedding/lm_head)
- QK scaling after norm: **none** >> q*sqrt(d) (near-uniform attention is intentional)
- WD masking: **cautious** >> standard (directional masking aids optimization)
