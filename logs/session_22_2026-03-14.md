# Session 22 — 2026-03-14

## Experiment

**Increase x0_lambda init from 0.1 to 0.15 for stronger skip connections**

Rationale: The x0_lambdas are learnable per-layer scalars controlling how much of the original embedding (x0) is mixed into the residual stream at each layer. A higher initial value (0.15 vs 0.1) strengthens the skip connection to the original token embeddings, potentially improving gradient flow through the network and providing a stronger identity signal. In short 5-minute training runs, initialization values of learnable parameters matter significantly since there's limited time to adapt.

## Changes

- `x0lambdas.fill()`: 0.1 → 0.15 (in `init_weights`)

## Analysis

- Increasing x0_lambda init from 0.1 to 0.15 hurt performance (1.0387 → 1.0484, +0.0097)
- Stronger initial skip connections dilute the residual stream with too much raw embedding signal
- The model needs room to develop its own representation hierarchy; 0.1 provides enough gradient flow without overwhelming layer outputs
- The x0_lambdas are learned parameters, so the optimizer could in principle adjust them, but with only ~1340 steps the initialization bias persists throughout training
- **Conclusion: x0_lambda init=0.1 is well-calibrated; higher values introduce too much embedding bypass and hurt layer specialization**
