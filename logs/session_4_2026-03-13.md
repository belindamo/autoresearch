# Session 4 — 2026-03-13

## Experiment

**Increase embedding LR from 0.6 to 1.0**

Rationale: Embeddings are a large fraction of the model parameters. With our smaller batch size giving more optimizer steps, a higher embedding LR could help them learn faster within the fixed 5-minute budget.

## Changes

- `EMBEDDING_LR`: 0.6 → 1.0

## Results

| Metric | Value |

|--------|-------|

| val_bpb | **1.0458** ✓ (prev best: 1.0472) |

| Peak VRAM | 22.7 GB |

| Steps | 1348 |

| MFU | 14.12% |

| Training seconds | 300.1 |

| Total tokens | 176.7M |

**Status: KEEP** — small but real improvement (−0.0014).

## Analysis

- The improvement is modest (0.0014) but the change is trivial (one number)
- Steps and VRAM identical to previous run, confirming no computational overhead
- This suggests embeddings were slightly under-learning at LR 0.6
- The simplicity criterion favors keeping this — zero complexity cost for a positive signal
