# Session 29 — 2026-03-15

## Experiment

**Change window pattern from "SSSL" to "SSL" — more full-context attention layers**

Rationale: With "SSSL" pattern and 8 layers, only 2 layers use full-context attention (25%). Switching to "SSL" gives 3 full-context layers (~37.5%), potentially improving long-range dependency modeling. The extra attention compute is modest (~10% more attention FLOPs) since sliding window is half the sequence length.

Note: The first attempt ran on the wrong branch (master, not autoresearch/sundial) due to missing `--branch` flag in `runonmodal.py`. That result (val_bpb=1.1179) was discarded as it used master's config (larger batch size 2^19, only 352 steps). The experiment was re-run correctly on the autoresearch/sundial branch.

## Changes

- `WINDOWPATTERN = "SSSL"`* → *`WINDOWPATTERN = "SSL"`

With 8 layers, this changes the pattern from:

- SSSL: S,S,S,L,S,S,S,L → 2 full-context layers (25%)
- SSL: S,S,L,S,S,L,S,L(forced) → 3 full-context layers (37.5%)

## Analysis

- The throughput was nearly identical to baseline (1358 steps vs typical ~1340), confirming the ~10% attention FLOPs increase is negligible
- Despite getting the same number of training steps, more full-context layers didn't improve val_bpb
- The result (1.0492) matches session 28's no-op result, suggesting this is approximately neutral within GPU variance
- The "SSSL" pattern with 25% full-context layers is well-calibrated — the model doesn't benefit from more global attention at seq_len=2048
- This is likely because at 2048 tokens, even the sliding window (1024) captures sufficient context for most patterns
- Window pattern map: **SSSL→best**, SSL→1.0492 (neutral/slight regression)
