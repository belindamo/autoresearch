### Experiment 4: Cosine warmdown schedule (replace linear)

**Rationale:** Cosine decay maintains higher LR longer in early warmdown and decays faster at the end — standard practice in LLM training.

| Metric | Value |

|--------|-------|

| val_bpb | **1.0476** ✗ (prev best: 1.0387) |

| Steps | 1346 |

| MFU | 14.09% |

| Tokens | 176.4M |

**Status: DISCARD** — val_bpb regressed by 0.0089. With 67% warmdown ratio, cosine shape keeps LR too high for too long. Linear decay provides more gradual decrease that's better for convergence in this regime.

### Experiment 5: SiLU activation (replace ReLU²)

**Rationale:** SiLU (Swish) is used in LLaMA and many modern architectures. Same param count, same structure — just a different activation function.

| Metric | Value |

**Status: DISCARD** — val_bpb regressed by 0.0101. ReLU² provides better sparsity properties than SiLU, which helps generalization even in short training. The squared ReLU creates naturally sparse activations that are beneficial for this model size.

## Analysis

- All six experiments regressed, confirming the model is deeply optimized for this time budget
- Typical GPU allocation produces ~1340 steps at 14% MFU, giving val_bpb in the 1.04-1.05 range
- The best result (1.0387, session 15) benefited from a faster GPU allocation (1560 steps, 16.35% MFU)
- GPU variance accounts for ~0.01 bpb, meaning the true optimum is likely 1.035-1.040
- Key confirmed optimal values this session:
    - Adam beta1: 0.7→1.0437, **0.8**, 0.9→1.0517
  - HEAD_DIM: **128** > 64

  - Activation: **ReLU²** > SiLU
