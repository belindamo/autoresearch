# Session 33 — 2026-03-16

## Experiments

Six experiments tested in this session. All regressed; best val_bpb unchanged at **1.0387**.

### Experiment 1: SwiGLU MLP (replacing ReLU² 4x MLP)

**Rationale:** SwiGLU is the standard MLP in LLaMA/Gemma/Mistral. Uses explicit gating (sigmoid gate × up projection), different from plain SiLU tested in session 31. Param-matched with hidden_dim=1408.

| Metric | Value |
|--------|-------|
| val_bpb | **1.0679** ✗ (prev best: 1.0387) |
| Steps | 1275 |
| MFU | 13.52% |
| VRAM | 23,077 MB |
| Tokens | 167.1M |
| Params | 50.9M |

**Status: DISCARD** — val_bpb regressed by 0.029. Despite being param-matched, SwiGLU is significantly worse. ReLU² sparsity properties are critical for this model size and training budget. The gating mechanism doesn't compensate. Confirms: **ReLU² > SiLU > SwiGLU** for this regime.

### Experiment 2: GQA n_kv_head=2 (grouped-query attention)

**Rationale:** Reduce KV projection parameters (50% fewer c_k, c_v params), trade model capacity for throughput. 4 query heads share 2 KV heads.

| Metric | Value |
|--------|-------|
| val_bpb | **1.0437** ✗ (prev best: 1.0387) |
| Steps | 1610 |
| MFU | 16.0% |
| VRAM | 20,912 MB |
| Tokens | 211.0M |
| Params | 39.8M |

**Status: DISCARD** — val_bpb regressed by 0.005. Model is 21% smaller (39.8M vs 50.3M) and 20% faster (1610 vs ~1340 steps), but the throughput gain doesn't compensate for reduced KV capacity. GQA is too aggressive with only 4 query heads.

### Experiment 3: Weight decay 0.05→0.04

**Rationale:** Fine-tuning WD between 0.025 (regressed) and 0.05 (current best). Session 32 attempted WD=0.04 but had infra timeout.

| Metric | Value |
|--------|-------|
| val_bpb | **1.0471** ✗ (prev best: 1.0387) |
| Steps | 1440 |
| MFU | 15.08% |
| Tokens | 188.7M |

**Status: DISCARD** — val_bpb regressed by 0.008. Note: highly variable step times (200-470ms) suggesting GPU contention. WD map completed: 0.025→1.0437, 0.04→1.0471, **0.05→1.0387**, 0.1→1.0420.

### Experiment 4: EMA decay 0.99→0.98

**Rationale:** More responsive weight averaging for evaluation. Previous test of 0.99→0.995 regressed; testing the opposite direction.

| Metric | Value |
|--------|-------|
| val_bpb | **1.0500** ✗ (prev best: 1.0387) |
| Steps | 1316 |
| MFU | 13.78% |
| Tokens | 172.5M |

**Status: DISCARD** — val_bpb regressed by 0.011. EMA map: 0.98→1.0500, **0.99→best**, 0.995→regressed. EMA=0.99 confirmed optimal from both directions.

### Experiment 5: torch.compile mode='reduce-overhead' (CUDA graphs)

**Rationale:** CUDA graphs reduce kernel launch overhead for fixed-shape models. Different from 'max-autotune' (tested session 31) which has autotuning overhead.

| Metric | Value |
|--------|-------|
| val_bpb | **1.0449** ✗ (prev best: 1.0387) |
| Steps | 1498 |
| MFU | 15.70% |
| Tokens | 196.3M |
| Total seconds | 604.7 |

**Status: DISCARD** — val_bpb regressed by 0.006. Throughput improved ~12% (1498 vs ~1340 steps, 202ms/step vs 232ms/step), but ~100s longer startup due to CUDA graph capture. The throughput gain is real but doesn't overcome the model's fundamental convergence limit at this step count.

### Experiment 6: Muon initial momentum 0.85→0.90

**Rationale:** Higher starting momentum for smoother early optimization. Session 23 tested lowering the target (0.95→0.90, regressed), but initial momentum was unexplored.

| Metric | Value |
|--------|-------|
| val_bpb | **1.0524** ✗ (prev best: 1.0387) |
| Steps | 1362 |
| MFU | 14.26% |
| Tokens | 178.5M |

**Status: DISCARD** — val_bpb regressed by 0.014. Higher initial momentum is too rigid, prevents exploration in early training. Momentum start map: **0.85→best**, 0.90→1.0524. The gradual ramp (0.85→0.95 over 300 steps) is important for early gradient exploration.

## Analysis

- All six experiments regressed, confirming the model remains deeply optimized for the 5-min time budget
- **Architecture changes (SwiGLU, GQA) regressed more than hyperparameter tweaks** — the current architecture is well-suited for this regime
- **Throughput improvements (reduce-overhead, GQA) don't compensate for quality loss** — more steps alone won't beat the current config
- **Fine-grained hyperparameter sweeps (WD=0.04, EMA=0.98, momentum=0.90) all regressed** — we're at local optima for most parameters
- Key confirmed optimal values this session:
  - Activation: **ReLU²** > SiLU > SwiGLU
  - Attention: **MHA (n_kv_head=4)** > GQA (n_kv_head=2)
  - Weight decay: 0.025→0.04→0.05(**best**)→0.1
  - EMA: 0.98→**0.99**→0.995
  - Muon momentum start: **0.85**→0.90
  - torch.compile: **default** > reduce-overhead > max-autotune
