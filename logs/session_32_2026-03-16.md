### Experiment 2: Weight tying (share wte and lm_head)

**Rationale:** Well-established technique (GPT-2, BERT, T5) that saves ~33% of parameters by sharing input embeddings and output projection. Fewer parameters = better generalization in short training.

| Metric | Value |
|--------|-------|
| val_bpb | **4.680841** ✗ (catastrophic) |
| Steps | 1500 |
| MFU | 14.07% |
| VRAM | 22,734 MB |
| Tokens | 196.6M |
| Params | 46.1M |

**Status: DISCARD** — Catastrophic failure. The model barely learned (loss stuck at ~8.2). Root cause: the embedding LR (1.0) and unembedding LR (0.004) differ by 250x. With weight tying, the shared weight gets the embedding LR (1.0), which is far too aggressive for the output projection role. The dual-LR design is fundamentally incompatible with weight tying.

### Experiment 3: Z-loss regularization (PaLM-style, coefficient 1e-4)

**Rationale:** Z-loss adds a soft penalty on raw logit magnitudes (logsumexp²), complementing the softcap by keeping logits in a well-behaved range. Used successfully in PaLM and other large models to improve gradient flow.

| Metric | Value |

**Status: DISCARD** — val_bpb regressed by 0.019. The softcap already provides sufficient logit regularization; the additional z-loss distorts the gradient landscape. The coefficient 1e-4 adds ~0.014 penalty (0.5% of cross-entropy loss), which is enough to hurt convergence without providing meaningful stabilization benefit.
