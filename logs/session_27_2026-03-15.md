# Session 27 — 2026-03-15

## Experiment

**Increase RoPE base frequency from 10000 to 50000**

Rationale: Higher RoPE base frequencies spread positional information across more dimensions and reduce the rate at which nearby positions become distinguishable, potentially helping the model learn longer-range dependencies. With seq_len=2048, the standard base=10000 may be over-emphasizing local position differences. This is a zero-complexity change (single constant).

## Changes

- `precomputerotary_embeddings(..., base=10000, ...)` → `base=50000`

## Results

| Metric | Value |

|--------|-------|

| val_bpb | **1.0405** ✗ (prev best: 1.0387) |

| Peak VRAM | 22,653 MB |

| Steps | 1,506 |

| MFU | 15.78% |

| Training seconds | 300.2 |

| Total tokens | 197.4M |

| Params | 50.3M |

**Status: DISCARD** — val_bpb 1.0405 vs best 1.0387, regressed by 0.0018; reverted.

## Analysis

- RoPE base=50000 gave 1.0405, which is only 0.0018 worse than the best (1.0387)
- This is a very marginal regression — within the range of GPU variance we've observed (session 15's best was on a faster GPU at 16.35% MFU vs typical ~14%)
- This run had good throughput (1506 steps, 15.78% MFU) — better than many previous runs
- The near-neutral result suggests the model isn't very sensitive to RoPE base at seq_len=2048 — both 10000 and 50000 work similarly
- May be worth trying an intermediate value (e.g., 20000 or 30000) or lower (e.g., 5000) in future, but the return is likely marginal
- Final train loss (~2.904) is similar to the best runs, suggesting convergence quality is comparable

## Next ideas

- Try Adam beta1 0.8→0.7 — even lower momentum for faster adaptation in short training
- Try lower scalar LR (0.5→0.3) — less aggressive per-layer gating
- Try Muon ns_steps 5→6 — more orthogonalization steps for better gradient conditioning
- Try n*kv*head < n_head (GQA, e.g., 2 kv heads with 4 query heads) — more efficient attention
- Try wider MLP (4x→6x hidden) with fewer layers — trade depth for width
- Try window pattern "SSL" — shorter pattern cycle, more full-context layers
- Try removing value embeddings entirely (simplification test) — see if ve_gate adds enough value
- Try compound: RoPE base=50000 + another marginal improvement to push past noise floor
